# 自修循环规则

## 触发条件

任意测试失败（包括回归失败）时启动自修循环。

## 循环规则

最多 5 轮，每轮必须输出：

```
Round N/5
- Failing tests: <失败测试列表>
- Root cause: <根因一句话>
- Changes: <文件 + 改动点，≤3文件>
- Re-run: PASS/FAIL + 摘要
```

## 修复优先级

1. **优先修实现**：测试是对的，实现有问题
2. **其次修测试**：测试写错了（极少数情况）

## 超限处理

**超过 5 轮仍不绿：立即停止**

输出：
```
[人工决策点]
- 已尝试: 5轮
- 仍失败: <列表>
- 疑似根因: <分析>
- 建议: <需要人工介入的原因>
```

目的：防止 AI 在错误方向上狂奔，扩大损害。

## 禁止行为

| 禁止         | 原因         |
| ------------ | ------------ |
| 删除失败测试 | 掩盖问题     |
| 放宽断言条件 | 降低质量标准 |
| 跳过测试     | 逃避问题     |
| 跨层乱改     | 扩大影响范围 |

## 变更围栏

每轮修复最多改 **3 个文件**。

超出范围必须：
1. 停止当前修复
2. 输出 ADR-lite
3. 等待人工批准
