# 门禁落地指南

## 必须至少选一个

### 方案一：本地 pre-push（推荐）

在 `.git/hooks/pre-push` 中：

```bash
#!/bin/bash
set -e

echo "=== Gate A: Parity Check ==="
python scripts/atdd_gate.py --plan TEST_PLAN.md --tests-root tests/atdd --parity-only
if [ $? -ne 0 ]; then
    echo "❌ Gate A failed: 清单与测试不一致"
    exit 1
fi

echo "=== Running Tests ==="
pytest tests/atdd --junitxml=test-results/junit.xml
if [ $? -ne 0 ]; then
    echo "❌ Tests failed"
    exit 1
fi

echo "=== Gate B: Auto-tick ==="
python scripts/atdd_gate.py --plan TEST_PLAN.md --junit test-results/junit.xml --tick --strict
if [ $? -ne 0 ]; then
    echo "❌ Gate B failed: 有未通过的验收条目"
    exit 1
fi

echo "✅ All gates passed"
```

### 方案二：CI Gate（GitHub Actions）

```yaml
name: ATDD Gate
on: [push, pull_request]

jobs:
  gate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: Gate A - Parity Check
        run: python scripts/atdd_gate.py --parity-only
      
      - name: Run Tests
        run: pytest tests/atdd --junitxml=test-results/junit.xml
      
      - name: Gate B - Tick Check
        run: python scripts/atdd_gate.py --tick --strict
```

配合 Branch Protection：
- 要求 PR 必须通过 CI
- 任一步失败 → 禁止合并

## 门禁规则

| 步骤            | 失败时                |
| --------------- | --------------------- |
| Gate A (parity) | push 失败             |
| 测试运行        | push 失败             |
| Gate B (tick)   | push 失败             |
| lint/typecheck  | push 失败（如有配置） |
