# 同名测试规则

## 核心规则

**测试用例的显示名/描述 = 清单条目文字（逐字一致）**

这是 Gate 机器对账的基础。名字不一致 = 暗号失效 = Gate 拦截。

## 各框架写法

### JavaScript/TypeScript (Jest/Mocha/Vitest)

```js
// ✅ 正确
test('用户输入错误的Email格式', () => {
  // ...
});

// ❌ 错误
test('测试Email报错', () => {
  // ...
});
```

### Go

```go
// ✅ 正确
t.Run("用户输入错误的Email格式", func(t *testing.T) {
    // ...
})
```

### Java (JUnit 5)

```java
// ✅ 正确
@Test
@DisplayName("用户输入错误的Email格式")
void testInvalidEmail() {
    // ...
}
```

### Python (PyTest)

```python
# ✅ 正确 - 用 docstring 放条目文字
def test_invalid_email():
    """用户输入错误的Email格式"""
    # ...
```

## 常见错误

| 错误类型 | 示例                                            |
| -------- | ----------------------------------------------- |
| 重写措辞 | 清单"登录失败显示错误" → 测试"测试登录错误提示" |
| 多余空格 | 清单"用户 登录" → 测试"用户登录"                |
| 标点差异 | 清单"..." → 测试"…"                             |
| 省略部分 | 清单"用户输入空密码时显示错误" → 测试"空密码"   |

## 验证方法

运行 Gate A 检查：

```bash
python scripts/atdd_gate.py --plan TEST_PLAN.md --tests-root tests/atdd --parity-only
```

输出 `false` 时会告诉你哪些不匹配。
