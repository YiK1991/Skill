# Jules Tools CLI — Quick Reference\n\n> 本页聚焦"能跑起来"的命令。可靠集成请使用 `scripts/dispatch_prompt_pack.py`（门禁自动化 + 稳定 JSON 输出）。\n\n## Installation\n\n```bash\nnpm install -g @google/jules\n# or\npnpm install -g @google/jules\n```\n\n## Authentication\n\n```bash\njules login\njules logout\n```\n\n## Version\n\n```bash\njules version\n# 注意：是子命令，不是 --version flag\n```\n\n## Commands\n\n### TUI Dashboard（交互模式）\n\n```bash\njules\n```\n\n> ⚠ **非交互终端（如 AI Agent 调用）中无法正常显示**。\n> Jules 的 TUI 基于 bubbletea，在非交互 stdout 中输出会乱码。\n> AI Agent 应使用 `jules remote` 子命令或 dispatch 脚本。\n\n### List repos / sessions\n\n```bash\njules remote list --repo\njules remote list --session\n```\n\n> ⚠ `--session` 输出包含 TUI 格式化字符，在非交互终端中可能截断。\n> 需要精确 session 状态时，使用 Bridge 的 `status` 命令。\n\n### Create a session\n\n```bash\n# 从 stdin 读取 prompt（Windows 推荐方式 — 仅人类手动/应急）\ncmd /c \"chcp 65001 >nul && type C:\\temp\\task.md | jules remote new --repo owner/repo\"\n\n# 直接指定 session 描述\njules remote new --repo owner/repo --session \"write unit tests\"\n\n# 从 cwd 自动检测 repo\njules remote new --repo . --session \"fix the login bug\"\n```\n\n> ⚠ `--parallel` flag 已知导致 API 400 错误（见 P7），**不建议使用**。\n\n### Pull results\n\n```bash\njules remote pull --session <session_id>\n```\n\n> ⚠ **PR-Only 工作流中禁止使用此命令**。所有结果通过 PR 审阅。\n\n## Bridge Script（非 submit 命令的集成接口）\n\n> **submit 命令已被硬拦截**：必须通过 `dispatch_prompt_pack.py` 提交任务。\n> Bridge 仅用于 status / wait / approve / send / tail 等非提交操作。\n> 如需调试 submit，必须设置 `_JULES_DISPATCH=1`（仅限调试）。\n\n```bash\n# 查询状态\npython jules_bridge.py --json status --session-id <id>\n\n# 等待状态变化\npython jules_bridge.py --json wait --session-id <id> --until COMPLETED,FAILED\n\n# 审批计划\npython jules_bridge.py --json approve --session-id <id>\n\n# 发送追加消息\npython jules_bridge.py --json send --session-id <id> --message-file followup.md\n\n# 尾部活动\npython jules_bridge.py --json tail --session-id <id> --limit 5\n```\n\n> ⚠ **参数顺序陷阱**：`--json`、`--repo` 等全局选项必须放在子命令**之前**。\n> ✅ 正确：`jules_bridge.py --json status --session-id <id>`\n> ❌ 错误：`jules_bridge.py status --session-id <id> --json`（会报 unrecognized arguments）\n\n## 提交任务（Submit — 唯一生产入口）\n\n```bash\nexport JULES_API_KEY=\"your-key\"\npython scripts/dispatch_prompt_pack.py \\\n  --pack-dir <path/to/jules_pack> \\\n  --repo <owner/repo> \\\n  --starting-branch master\n```\n\n## Windows 特别注意\n\n1. `cmd /c` 管道方式是 Windows 上 CLI 直接调用的兜底方式（但应优先用 dispatch）\n2. 必须先 `chcp 65001` 设为 UTF-8，否则中文 prompt 会乱码\n3. PowerShell 中调用 bridge 时，路径反斜杠可能引发解析问题，建议用 `cmd /c` 包裹\n4. `jules version` 是子命令（不是 `--version` flag）\n
