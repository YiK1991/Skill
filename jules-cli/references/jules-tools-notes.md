# Jules Tools (CLI) Notes\n\n> CLI 仅适用于人类手动操作或应急兜底。程序化集成首选 `dispatch_prompt_pack.py`。\n\n---\n\n## CLI 常用命令（脚本化）\n\n- 版本：`jules version`（注意：是子命令，不是 `--version`）\n- 登录：`jules login`\n- 列表：`jules remote list --repo` / `jules remote list --session`\n- 创建 session（人类应急）：`cmd /c \"chcp 65001 >nul && type file.md | jules remote new --repo owner/repo\"`\n- 拉取代码：`jules remote pull --session <session_id>`（PR-Only 流中禁止使用）\n\n> ⚠ `echo \"...\" | jules remote new` 存在 pipe 编码风险（P1/P3），应使用 `type file |` 方式。\n> ⚠ `--parallel` flag 已知导致 API 400 错误（见 P7），**不建议使用**。\n\n---\n\n## TUI vs Non-Interactive 模式\n\n| 调用方式               | 行为                         | AI Agent 可用？      |\n| ---------------------- | ---------------------------- | -------------------- |\n| `jules`（无参数）      | 启动 bubbletea TUI dashboard | ❌ 在非交互终端中乱码 |\n| `jules remote list`    | 返回结果但含 TUI 格式字符    | ⚠ 可用但输出可能截断 |\n| `jules remote new ...` | 创建 session，返回 ID        | ✅ 可靠               |\n| `jules version`        | 返回版本号                   | ✅ 可靠               |\n| dispatch_prompt_pack   | 稳定 JSON 输出 + 门禁        | ✅ 首选               |\n\n**结论**：程序化集成首选 `dispatch_prompt_pack.py`。AI Agent 应使用 Bridge API 模式（status/wait/approve/send）。\nCLI `jules remote` 子命令仅适用于人类手动操作或应急场景。\n\n---\n\n## Bridge Script 参数顺序陷阱\n\nBridge 的全局选项（`--json`、`--repo`、`--mode` 等）必须放在子命令之前：\n\n```bash\n# ✅ 正确\npython jules_bridge.py --json status --session-id <id>\n\n# ❌ 错误（报 unrecognized arguments）\npython jules_bridge.py status --session-id <id> --json\n```\n\n原因：argparse 将 `--json` 注册在根 parser 上，子命令有独立的 subparser。\n\n---\n\n## CLI 对接的常见坑\n\n1. **stdout 非稳定**：版本更新后输出可能变化；不要把 stdout 当协议。\n2. **session_id 解析不可靠**：部分输出可能不包含 id；应做二次验证。\n3. **重试导致重复 session**：必须做 idempotency（dispatch 已内置）。\n4. **缺少状态机信号**：CLI 无法稳定判断何时需要 plan approval / feedback（需 API 模式）。\n5. **TUI 乱码**：`jules` 和 `jules remote list --session` 在非交互终端中输出包含 bubbletea 渲染序列。\n6. **PowerShell 路径问题**：在 PowerShell 中反斜杠路径可能引发解析错误，用 `cmd /c` 包裹更可靠。\n7. **`--parallel` flag**：已知在某些场景下导致 API 400 错误，不建议使用。\n\n**因此**：CLI 只建议作为"本地人类操作"或"兜底路径"。程序化集成首选 `dispatch_prompt_pack.py`。\n
